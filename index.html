Bibliothèque IDF, Spring Boot DevOps

Ce projet académique consistait à développer le back-end d'une plateforme de gestion pour la Bibliothèque IDF, en équipe avec Nada Boussetta. J'ai travaillé sur une architecture Java Spring Boot exposant une API REST, avec un système d'échange en temps réel pour notifier les utilisateurs (nouvelle disponibilité, retour de livre) et une gestion de sessions pour distinguer les profils lecteurs et bibliothécaires. La partie la plus technique portait sur les algorithmes de recherche et d'optimisation : il fallait croiser plusieurs bibliothèques du réseau IDF pour retrouver rapidement un livre disponible, en tenant compte de la localisation et du délai de réservation. This academic project involved building the back-end of a management platform for Bibliothèque IDF, working as a team with Nada Boussetta. I worked on a Java Spring Boot architecture exposing a REST API, with a real-time messaging system to notify users (new availability, book returns) and session management to separate reader and librarian profiles. The most technical part dealt with search and optimization algorithms: the system had to cross-check several libraries in the IDF network to quickly find an available book, factoring in location and reservation delay. The project also included a basic DevOps pipeline, Git versioning and continuous integration, to make team deployments more reliable.

Analyse de données sur graphes et IA

Graph Data and AI Analysis

Ce projet portait sur l'étude de l'expression de gènes chez des souris mutées, à partir d'un jeu de données Open Data conséquent : 771 pathways, 4 980 gènes et 2 121 colonnes. Après une phase de nettoyage et de structuration des données sous Python avec Pandas, j'ai construit un réseau de 731 nœuds et 32 355 arêtes représentant les interactions entre gènes. L'analyse s'est ensuite concentrée sur deux axes : le calcul de métriques de centralité pour repérer les gènes les plus influents dans le réseau, et la détection de communautés pour faire apparaître des groupes de gènes qui interagissent fortement entre eux. Ce projet m'a fait sortir du cadre purement applicatif pour aller vers l'analyse scientifique de données à grande échelle.

This project studied gene expression in mutated mice, based on a substantial open data set: 771 pathways, 4,980 genes and 2,121 columns. After cleaning and structuring the data in Python with Pandas, I built a network of 731 nodes and 32,355 edges representing gene interactions. The analysis then focused on two angles: computing centrality metrics to spot the most influential genes in the network, and running community detection to surface groups of genes that interact strongly with each other. The project pushed me beyond pure application work into scientific analysis of large-scale data.

Étude des tarifs TGV, SNCF (TGV INOUI / OUIGO)

TGV Fare Study, SNCF (TGV INOUI / OUIGO)

Ce projet Power BI visait à comparer les grilles tarifaires de TGV INOUI, OUIGO et OUIGO Train Classique, à partir du jeu de données officiel publié par la SNCF sur data.gouv.fr, soit 36 349 trajets. La première étape a consisté à nettoyer et structurer les données brutes avec Power Query : correction des types, suppression des colonnes inutiles, remplacement des valeurs incohérentes. J'ai ensuite construit un modèle en étoile avec une table de faits reliée à quatre tables de dimension (gares, transporteurs, classes, profils tarifaires), ce qui a permis de croiser facilement les indicateurs de prix par gare, par transporteur ou par profil tarifaire. Plusieurs tableaux de bord interactifs présentent les résultats : répartition des trajets par transporteur, carte des gares desservies, comparaison des prix moyens minimum et maximum, écarts liés au yield management, et positionnement des trois transporteurs sur la vitesse et le confort.

This Power BI project set out to compare the fare structures of TGV INOUI, OUIGO and OUIGO Classic, using the official SNCF dataset published on data.gouv.fr, covering 36,349 trips. The first step was cleaning and structuring the raw data with Power Query: fixing data types, removing unnecessary columns, replacing inconsistent values. I then built a star schema with a fact table linked to four dimension tables (stations, carriers, travel class, fare profiles), which made it easy to cross-reference price indicators by station, carrier or fare profile. Several interactive dashboards present the results: trip distribution by carrier, a map of served stations, a comparison of average minimum and maximum prices, gaps tied to yield management, and how the three carriers compare on speed and comfort.
